SafeCurrent Setup Tutorial
==========================

This project has a Python backend and a simple HTML frontend.

The backend uses these Python packages:

- fastapi: creates the API server and the /simulate endpoint
- uvicorn: runs the FastAPI backend locally
- copernicusmarine: connects to Copernicus Marine data
- pandas: works with tables and timestamps
- numpy: handles drift/vector calculations
- requests: calls the Open-Meteo fallback API


Why create a virtual environment?
---------------------------------

A virtual environment is an isolated Python environment for this project.

It is recommended because:

- It keeps this project's packages separate from other Python projects.
- It avoids version conflicts between projects.
- It makes the project easier to reproduce on another computer.
- It keeps your global Python installation cleaner.

It is not strictly required. If you skip it, pip will install the packages into
your global/user Python environment instead. For quick hackathon work that can
be okay, but using a virtual environment is the cleaner and safer option.


1. Clone the repository
-----------------------

Open PowerShell and run:

git clone https://github.com/AdiVolfman/CS_Hackaton_26.git
cd CS_Hackaton_26


2. Create a virtual environment
-------------------------------

Recommended:

python -m venv .venv

This creates a folder named .venv that contains the isolated Python environment.


3. Activate the virtual environment
-----------------------------------

In PowerShell:

.venv\Scripts\Activate.ps1

After activating it, your terminal prompt may show (.venv).

If PowerShell blocks the activation script, run:

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Then try activating again:

.venv\Scripts\Activate.ps1


4. Install the requirements
---------------------------

pip install -r requirements.txt


5. Log in to Copernicus Marine
------------------------------

Run:

copernicusmarine login

Then enter your Copernicus Marine username and password when prompted.

You only need to do this once on your computer unless the credentials expire or
you want to change accounts.


6. Run the backend server
-------------------------

python -m uvicorn main:app --reload --port 8000

Keep this terminal open while using the app.

The backend should run at:

http://127.0.0.1:8000

You can test the API in the browser with an example like:

http://127.0.0.1:8000/simulate?lat=32.0800&lon=34.7650&hours=4


7. Open the frontend
--------------------

Open index.html in your browser.

The frontend sends requests to:

http://127.0.0.1:8000/simulate

So the backend must be running before the map can calculate drift.
